- 🏆 **Peak Performance — "best settings" finder (on top of the real backtester).** Given an MA-cross alert, sweep every fast/slow MA pair, run the **real** `runMaCrossBacktest` per combo, then rank the *full qualifying pool* by the client's chosen objective — **💰 Profit / 🎯 Accuracy / ⚖️ Balanced** — with an adjustable **min win-rate floor**. The core (`apps/api/src/optimizer.ts` `rankPeak`, all helpers pure + unit-tested) applies HARD pre-filters so the flashy fluke is *removed, not down-ranked*: min-trades (default 10), the win-rate floor, must-have-a-losing-trade (excludes the suspicious zero-loss/∞-PF combos), positive expectancy + PF≥1 for profit/balanced, max-drawdown cap. Survivors are enriched with `expectancyPct`, **`profitFactorCapped` (Infinity→5, never leaks to the wire/comparator)**, a 0–1 `qualityScore` (Sharpe·expectancy·PF·hit-rate − DD penalty, scaled by sample-size confidence), and neighbour-based **robustness flags** (lone-spike / deep-DD / few-trades / low-PF / robust — *zero extra backtests*, computed from the grid it already ran). Sort is per-objective with **deterministic config-key tiebreaks** (Apply writes to a live alert, so order must be reproducible). The legacy optimizer path (no `objective`) is preserved byte-for-byte.
